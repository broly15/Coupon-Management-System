# Development Rules

Always complete the current sprint before starting the next.

Every sprint should finish with:

- Clean architecture
- Responsive UI
- Production-quality code
- No placeholder code
- Reusable components
- TypeScript without errors

Never sacrifice maintainability for speed.

Always prioritize quality over quantity.
